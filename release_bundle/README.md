## Quick Start

### Step 1 — Generate the data file (one time only, ~2 minutes)

```bash
pip install scipy          # if not already installed
python generate_ppmi_proxy.py
```

This downloads WikiText-2 from the public source and builds a PPMI-SVD embedding.
The output is saved to `data/ppmi_proxy.npz` (~3–5 MB).

WikiText-2 source: Merity et al. 2016 — https://arxiv.org/abs/1609.07843  
Public download: https://s3.amazonaws.com/research.metamind.io/wikitext/wikitext-2-v1.zip

### Step 2 — Run the demo (numpy only, <60 seconds)

```bash
python hopf_routing_demo.py
```

Expected output:
- Power law fit: HOPF alpha ≈ 0.572, PERM alpha ≈ 0.814 (matches Table 3)
- Reference values: HOPF c=2.957 R²=0.963, PERM c=1.664 R²=0.993
- Compression ratios vs PERM at K=100 (≈1.9×) and K=400 (≈2.2×)

### Step 3 — Regenerate Figure 1 (optional, requires matplotlib)

```bash
pip install matplotlib
python generate_figures.py
# produces figures/fig_scaling_law.pdf
```

## Claim Scope

This bundle reproduces the **Tier 1 proxy result** (Sections 2–3):
- Fixed Hopf routing scales as K^0.572 vs K^1.0 for dense routing
- The advantage persists at K=5000 (ratio 2.6–2.8×, does not collapse)

The **Tier 2 LM result** (Section 4) — 8% PPL cost at 2-layer toy scale across PTB
and WikiText-2 — is established by INC-0171 and INC-0173 in the full research
record; the full analysis scripts are not included in this minimal bundle.

## Reproducibility Notes

- The routing algorithm is purely deterministic given `SEED=42` and the data file.
- Small deviations from Table 3 reference values (<0.5%) are expected due to
  random seed differences in PPMI construction if the data file is regenerated.
- If you regenerate `ppmi_proxy.npz`, the power law fit will use the same algorithm
  but may differ slightly from the archived numbers; this is expected and normal.
